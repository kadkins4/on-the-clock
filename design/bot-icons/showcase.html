Custom bot icons — three directions

Replacing the emojis for the 9 bot strategies (plus Normal). The choice that matters isn't the individual glyphs — it's the visual language. These render at ~18px in the mix picker and the end-of-draft reveal, so they must read instantly at that size. Emoji's one real edge is colour-coded scanning, so my lead recommendation keeps a per-strategy tint. Pick a direction; I'll refine the winning set to final art.

How it ships

Today each strategy carries an emoji string in its icon field. The clean swap: keep the field as a stable iconId and add one BotIcon component that maps id → inline SVG (an <svg><use> sprite, exactly like this page). The per-strategy tint becomes a --c CSS var already available where we render. Nothing about the engine, reveal, or picker logic changes — only the presentational glyph. One-file, fully reversible.
